# Refonte de la presentation Compound Engineering

## Contexte

Presentation de veille tech pour des coequipiers developpeurs chez Sfeir. Objectif : partager le concept de Compound Engineering et le situer dans le paysage actuel du developpement AI-native.

## Decisions prises

### Arc narratif : spectre de maturite

Presenter trois approches comme un spectre evolutif (pas des concurrents) :

1. **Vibe Coding** — "je sais pas trop ce que je veux, debrouille-toi"
2. **Spec Driven Development** — "je sais ce que je veux (WHAT/WHY), a toi de le construire (HOW)"
3. **Compound Engineering** — "je sais ce que je veux, tu le construis, on verifie ensemble, et on apprend pour la prochaine fois"

Le CE n'est pas une reponse au SDD, c'est une suite logique. Le SDD resout "comment bien briefer un agent". Le CE resout "comment faire en sorte que chaque brief soit meilleur que le precedent".

### Structure des slides

1. Le probleme commun (existant)
2. Le spectre Vibe Coding → SDD → CE (nouveau)
3. SDD : WHAT/WHY/HOW, outils (SpecKit, OpenSpec), forces et limites
4. CE comme evolution naturelle : la boucle, le split 80/20, les 5 etapes
5. Commandes cles (1 slide allege, pas de listes exhaustives)
6. Filesystem comme base de connaissances
7. Echelle d'adoption, croyances/principes
8. Resultats concrets, demarrage, ressources
9. One More Thing : atrophie cognitive — lie au spectre ("meme au stade CE, qui maintient VOS competences ?")
10. Call-to-action concret : 3 pratiques pour lundi

### Choix de design

- Supprimer le slide "Nouveautes 2026" — integrer les infos a jour dans les slides existants
- Supprimer le slide "Pratiques Emergentes" — pas assez ancre pour un talk de veille
- Alleger les commandes : 1 slide max, garder les plus parlantes, renvoyer vers ressources
- One More Thing : renforcer avec un appel a l'action concret (sessions sans IA, red-teaming, expliquer avant de merger)

## Non-goals

- Ce n'est pas une formation a l'utilisation du plugin
- Pas de demo live
- Pas de comparaison exhaustive des outils SDD
